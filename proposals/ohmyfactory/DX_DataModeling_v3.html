v3 · 2026-07-22 · 공통 7 + 마스터 8 + 업무 18 = 33테이블. enum은 시트별 관리, 결재는 approval 단독, 검사 가이드/테이블 변경이력 각 1개.

v2 → v3 변경 요약 · 검토 포인트 (펼침/접힘)

반영한 변경 (단순화)

  • enum 시트별 유지: 중앙 enum_set/enum_value 는 두지 않음. check_item.option_values 로 항목별 선택값 관리, 상태·판정·구분 필드는 시트 enum(코드값)으로 저장
  • 검사 가이드 1테이블: sheet_guide(시트별 가이드 내용 + 이미지 첨부). 업무 테이블의 sheet_guide_id 는 제거, 시트 단위로 참조
  • 결재는 approval 단독: document_type·approval_route 없이 기존 approval(다형 참조)로 처리. 결재 유무는 결재행 존재로 판단
  • 테이블 변경이력: table_change_log(전 테이블 변경 전/후값·시점 기록). 호기명 변경 등 과거 기록의 시점 보존용
  • MES 실적 수신 제외: 별도 수신 테이블·source_type_id 없음. 생산실적은 수기 입력 기준
  • 용어 통일: 대표 업무일자 → record_date
  • check_result: v2의 user_id 제거(created_by 일원화)
  • 07 사출조건 tonnage 제거 · 03 참고자료 item_name+is_used 롱포맷 · 30 resin_name/grade→material 연동 · 18 참가자 서명 구조화

검토(확인) 포인트

  • 테이블 변경 시점 보존: table_change_log(변경로그) 방식. 완전한 시점조회가 필요하면 SCD Type2(유효기간)로 전환 — 고객 확인 대상
  • 17 재검사 good_qty_after·defect_rate 는 파생(자동산출) 표기. 저장 대신 조회 계산 권장
  • 시트 enum(주야·판정 등)은 코드값을 시트별로 관리 — 여러 시트 공유가 잦아지면 공통화 재검토
  • 붉은 테두리 칩 = v3 신규/변경 필드
