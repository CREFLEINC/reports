데이터로 결정하는 스마트 제조

Multi-Agent 시스템으로 실시간 데이터 수집부터 상황 인지, 생산 전략 계획, 경영 의사결정 지원까지 — 제조·생산 비용을 최소화하고, 데이터 근거 기반의 빠르고 정확한 의사결정 환경을 구축합니다.

🏭 테스트베드 · 삼진엘앤디 🧩 플라스틱 사출 제조 📊 생산 제조 최적화 ⏱ 8주 PoC
01 — Overview

프로젝트 개요

Multi-Agent를 이용한 스마트 제조 공장 환경을 구축합니다. 스마트 공장에서 Multi-Agent는 실시간 데이터 수집, 실시간 상황 인지, 생산 전략 계획, 경영 의사결정 지원을 담당합니다.

프로젝트 명
Oh My Factory
프로젝트 단계
PoC — Multi-Agent 시스템의 가치를 검증할 수 있는 최소한의 가치 검증을 수행합니다.
기간
8주 (데이터 → Agent → MAS → 시연·운영 교육). ※ 검증 및 효과 측정은 8주 종료 이후 별도 일정으로 진행
테스트베드
고객사 삼진엘앤디의 플라스틱 사출품 제조 공장의 생산 제조 최적화를 진행합니다. 데이터만 가지고 가상 환경에서 예측하는 것이 아니라, 실제 제조 현장에서 Multi-Agent 시스템의 역할이 얼마나 가치가 있을 수 있는지 검증합니다.

왜 Multi-Agent 스마트 공장인가

  • 제조·생산 비용 최소화 — 생산 이익을 극대화할 수 있는 생산·제조 전략을 항상 유지합니다.
  • 365일 24시간 실시간 감시 — 모든 데이터를 실시간 감시하여, 긴급 상황 인지가 늦어져 발생할 수 있는 위험 손실 비용을 방지합니다.
  • 데이터 근거 기반 의사결정 — 언제 어디서든 경영자가 빠르고 정확하게 결정할 수 있는 환경을 구축합니다.
💡 숫자 + 경험

모든 일을 숫자만을 근거로 진행할 수는 없습니다. 그러나 "숫자"라는 근거 위에 경영진의 경험과 직관을 더한 의사결정은 위험을 낮추고 성공 확률을 극대화할 수 있습니다.

03 — Validation Scenarios

검증 시나리오

실제 제조 현장에서 반복되는 의사결정 상황을 PoC에서 재현하여 Multi-Agent 시스템의 가치를 검증합니다.

1
긴급 증산 요청 대응
고객사의 긴급 증산 요청 시, 라인 전환·야간 가동·외주 등 복수의 생산 옵션을 순이익과 리스크 기준으로 동시 비교하여 최적안을 추천하고, 근거가 포함된 의사결정 보고서를 자동 생성합니다.
다중 옵션 비교 → 최적안 추천 → 보고서 자동 생성
2
원자재 가격 급등 대응
원자재 가격이 급등할 때 영향을 받는 SKU와 적자로 전환되는 품목을 선제적으로 자동 식별하여, 담당자가 출근하기 전에 대응이 필요한 품목을 알립니다.
위험 감지 → 영향 SKU·적자 품목 자동 식별 → 선제 알림
3
신규 수주 기회 최적화
신규 견적·수주 기회가 발생했을 때 라인 가용성과 누적 순이익을 분석하여, 손실을 방지하고 이윤을 최적화하는 수주 구조를 설계합니다.
라인 가용성·누적 순이익 분석 → 최적 수주 구조 설계
04 — System Concept

Multi-Agent 시스템 개념

스마트 공장의 Multi-Agent는 네 가지 핵심 역할을 협력적으로 수행하며, 사용자는 자연어 질의와 대시보드를 통해 시스템과 상호작용합니다.

핵심 역할

📡
실시간 데이터 수집
생산·자재·주문·재고·원가 데이터를 실시간으로 수집합니다.
🔎
실시간 상황 인지
수집된 데이터에서 위험·기회 상황을 실시간으로 인지합니다.
🧭
생산 전략 계획
비용 최소화·이익 극대화 관점에서 생산·제조 전략을 수립합니다.
🎯
경영 의사결정 지원
근거 데이터와 함께 최적안을 제시하여 경영 의사결정을 지원합니다.

사용자 접점 — 자연어 질의 + 대시보드

사용자는 별도의 폼·그리드 입력 없이 자연어로 질문하고, 시스템은 대시보드로 다중 옵션 분석과 추천 근거를 시각화하여 답합니다.

🤝 Human-in-the-loop

Agent는 데이터를 분석하고 최적안을 제안하며, 최종 결정은 사람이 합니다. 모든 추천에는 근거 데이터가 기록되어 사후 검토와 추적이 가능합니다.

🧩 도구 개념

각 Agent는 사내 데이터·지식·다른 Agent와 연결하기 위해 세 가지 표준 도구를 사용합니다. MCP는 도구·데이터 연결, RAG는 지식 검색, A2A는 Agent 간 소통을 담당하며 서로 상호보완적입니다.

MCPModel Context Protocol

Agent ↔ 도구·데이터. MES·ERP 등 사내 시스템의 정형 데이터를 표준 방식으로 읽기. 기존 시스템을 대체하지 않고 그 위에서 연동합니다.

RAGRetrieval-Augmented Generation

Agent ↔ 지식. 1단계에서 벡터화한 비정형/문맥 데이터(영업일지·과거 의사결정 보고서)를 검색해, 수치로 담기 어려운 정성적 판단 근거를 의사결정에 반영합니다.

A2AAgent2Agent Protocol

Agent ↔ Agent. Google이 공개하고 Linux Foundation이 관리하는 개방형 표준. 각 Agent가 내부 로직·도구를 노출하지 않고(캡슐화) 작업을 위임·협력합니다.

기간 1주W8

개발된 Multi-Agent 시스템을 현장에 시연하고, 운영자가 직접 사용할 수 있도록 교육합니다.

  • 현장 시연 — 검증 시나리오(긴급 증산 / 원자재 급등 / 신규 수주)를 실제 데이터로 재현하여 시스템 동작을 시연합니다.
  • 운영 교육 — 자연어 질의·대시보드 사용법, 추천 근거 해석, 의사결정 적용 방법을 운영자에게 교육합니다.
📌 후속 · 검증 및 효과 측정 (8주 이후 별도 일정)

8주 PoC 종료 이후, 실제 운영 데이터를 기반으로 의사결정 시간 단축·순이익 개선·추적성 등 검증 지표의 실제 효과를 측정하는 단계를 별도 일정으로 진행합니다. 본 PoC 기간(8주)에는 시연 및 운영 교육까지만 포함됩니다.
