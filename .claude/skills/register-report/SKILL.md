# 신규 HTML 리포트 등록 (register-report)

외부 프로젝트에서 완성된 HTML 보고서를 **CREFLE Reports 서버에 등록**하는 절차다.
배치·동기화·반영확인의 결정적 작업은 번들 스크립트가 수행하고, 이 스킬은 **입력 수집과
자산(asset) 동반 여부 판단**을 담당한다.

## 언제 / 언제 아닌가

- **사용**: "이 리포트 등록해줘", "보고서 새로 올려줘", "버전 올려서 다시 반영", "reporter 에 추가"
- **사용 안 함**: 리포트 *내용 작성/디자인* 요청(외부에서 수행), 단순 목록 조회.

## 시스템 전제 (왜 이렇게 동작하나)

대상 프로젝트 `/Users/rangkim/projects/crefle/reports` 의 `server.py` 는 **매 요청마다
`proposals/` 를 스캔**해 목차를 만든다. hulk 컨테이너는 `proposals/` 를 bind mount(:ro)로
물고 있으므로, **파일을 호스트에 복사하면 재빌드·재시작 없이 즉시 반영**된다.
git repo 의 `proposals/` 가 **소스 오브 트루스**이고, hulk 에는 rsync 로 동기화한다.

## 입력 계약

등록 요청에서 다음을 받는다. 누락 시 사용자에게 되묻는다.

| 입력 | 필수 | 설명 | 예 |
|------|------|------|----|
| html 경로 | ✅ | 등록할 `.html` 파일 경로(외부 산출물) | `~/work/foo/report.html` |
| 문서 유형 | ✅ | `proposal` · `demo` · `ohmyfactory` · `<기타>` | `ohmyfactory` |
| 이름 | ✅ | 파일명에 쓸 이름(공백·한글 허용) | `삼진엘앤디_신규제안` |
| 버전 | ✅ | `1`, `2`, `0.1` (앞의 `v` 는 자동 제거) | `1` |
| 자산 디렉터리 | ⛔(조건부) | 리포트가 참조하는 로컬 자산 폴더(상대경로 자산이 있을 때) | `~/work/foo/assets-dir` |

**유형 → 폴더 매핑** (스크립트와 동일):
- `proposal` / `demo` → `proposals/` (루트, 목차 섹션 "제안서 · 데모")
- `ohmyfactory` → `proposals/ohmyfactory/` (섹션 "OhMyFactory (삼진엘앤디 DX)")
- 그 외 `<type>` → `proposals/<type>/` (신규 섹션, 라벨은 경로로 표시)
- **파일명 규칙**: `{이름}_v{버전}.html` (기존 관례와 동일)

## 절차

1. **입력 확인** — 위 4개 필수값을 확보한다. 없으면 한 번에 모아 되묻는다.
2. **자산 판단(핵심)** — 등록할 HTML 을 열어 상대경로 자산(`src=`/`href=` 중 `http`·`#`·`data:`
   가 아닌 것)을 확인한다.
   - 자산이 **이미 대상 폴더에 존재**(예: `ohmyfactory` 의 공유 `colors_and_type.css`·`assets/`·`fonts/`)하면 추가 조치 불필요.
   - 자산이 **없으면** 작성자에게 자산 폴더를 받아 `--assets <dir>` 로 함께 넣는다. 외부 CDN 만 쓰면 단일 파일로 충분.
3. **스크립트 실행** — 결정적 배치·동기화·확인:
   ```bash
   /Users/rangkim/projects/crefle/reports/.claude/skills/register-report/scripts/register_report.sh \
     --src "<html경로>" --type "<유형>" --name "<이름>" --version "<버전>" \
     [--assets "<자산폴더>"] [--commit] [--no-sync]
   ```
   스크립트가 하는 일: 유형→폴더 결정 → `{이름}_v{버전}.html` 로 배치 → (자산 병합) →
   `<title>`·상대자산 점검(경고) → (옵션)git commit → **hulk 접속 가능하면 rsync 동기화** →
   인코딩 URL 로 `200` 반영 확인. 내부망이 아니면 배치만 하고 수동 동기화 명령을 안내한다.
4. **결과 보고** — 사용자에게: repo 경로, 열람 URL(`http://192.168.1.111:28080/...`), 목차 URL,
   그리고 경고(자산 누락/제목 없음/동기화 불가)가 있으면 함께 전달한다.

## 옵션 판단

- **git 커밋**: repo 가 소스 오브 트루스이므로 `--commit` 권장. 푸시까지면 `--push`.
- **동기화 생략**: 내부망이 아닐 때 `--no-sync` 로 배치만 하고, 내부망 진입 후 재동기화한다.
  (스크립트는 hulk 접속을 자동 감지해 불가하면 안전하게 건너뛴다.)
- **덮어쓰기**: 같은 이름·버전이 이미 있으면 기본 거부 → 버전을 올리거나 `--force`.

## 신규 유형의 목차 섹션 이름(선택)

새 유형 폴더는 목차에 경로명으로 노출된다. 보기 좋은 섹션명이 필요하면 `server.py` 의
`GROUP_LABELS` 에 `"proposals/<type>": "표시명"` 을 추가한다. **이는 코드 변경이라
이미지 재빌드+재배포가 필요**하다(목차에 뜨는 것 자체는 라벨 매핑 없이도 동작).

## 실행 모드

단일 책임의 선형 절차이므로 **서브 에이전트(`report-registrar`)** 로 실행한다(팀 통신 불필요).
판단이 단순하면 메인에서 스크립트를 직접 실행해도 된다. 자산 동반·유형 신설 등 판단이
필요하면 `report-registrar` 에이전트에 위임한다.

## 테스트 시나리오

- **정상**: `--src 새보고서.html --type proposal --name 테스트보고서 --version 1` →
  `proposals/테스트보고서_v1.html` 배치 → hulk 동기화 → `…/proposals/테스트보고서_v1.html` 200.
- **에러(내부망 아님)**: hulk 접속 불가 → 배치만 수행하고 수동 rsync 명령 안내 후 종료(실패로 보지 않음).
- **에러(자산 누락)**: 상대 자산 참조가 대상 폴더에 없음 → 경고 출력, 등록은 진행하되 `--assets` 재실행 권고.
